자바에서는 정규식을 처리하기 위해 주로 **`java.util.regex`** 패키지를 사용합니다. 이 패키지는 **정규 표현식**을 다루기 위한 다양한 클래스와 인터페이스를 제공합니다. 여기서는 자바에서 정규식을 활용할 수 있는 주요 클래스, 메서드, 인터페이스를 정리하고 설명하겠습니다.

### 주요 클래스
`java.util.regex` 패키지의 핵심 클래스는 다음과 같습니다:

1. **`Pattern` 클래스**
   - 정규식을 표현하고 컴파일하는 데 사용됩니다.
   - 정규식 패턴을 해석하고, 이를 기반으로 문자열을 검색하거나 분석하는 데 쓰입니다.
   
   - 주요 메서드:
     - **`Pattern.compile(String regex)`**: 주어진 정규식을 컴파일하여 `Pattern` 객체를 생성합니다.
       ```java
       Pattern pattern = Pattern.compile("\\d+");
       ```
     - **`Pattern.matcher(CharSequence input)`**: 정규식을 기반으로 문자열을 매칭할 수 있는 `Matcher` 객체를 반환합니다.
       ```java
       Matcher matcher = pattern.matcher("12345");
       ```
     - **`Pattern.split(CharSequence input)`**: 정규식을 기준으로 문자열을 나누어 `String` 배열로 반환합니다.
       ```java
       String[] result = pattern.split("one,two,three");
       ```

2. **`Matcher` 클래스**
   - 정규식을 이용하여 문자열과의 매칭 작업을 수행하는 데 사용됩니다.
   - `Pattern` 객체에서 생성되며, 실제로 문자열과 매칭 작업을 수행하는 역할을 합니다.
   
   - 주요 메서드:
     - **`boolean matches()`**: 전체 문자열이 정규식과 일치하는지 여부를 반환합니다.
       ```java
       boolean isMatch = matcher.matches();
       ```
     - **`boolean find()`**: 문자열 내에서 패턴과 일치하는 부분이 있는지 찾고, 있으면 `true`를 반환합니다. 여러 번 호출하여 매칭되는 모든 부분을 탐색할 수 있습니다.
       ```java
       while (matcher.find()) {
           System.out.println(matcher.group());
       }
       ```
     - **`String group()`**: 마지막으로 매칭된 부분 문자열을 반환합니다.
     - **`int start()`**, **`int end()`**: 마지막으로 매칭된 부분의 시작 및 끝 인덱스를 반환합니다.
     - **`boolean lookingAt()`**: 문자열의 **시작 부분**이 패턴과 일치하는지 확인합니다.

3. **`PatternSyntaxException` 클래스**
   - 정규식의 문법이 잘못되었을 때 발생하는 예외를 처리하는 클래스입니다.
   - 정규식이 잘못된 경우 이 예외를 던져 오류의 위치 및 내용을 확인할 수 있습니다.
   
   - 주요 메서드:
     - **`String getDescription()`**: 오류에 대한 설명을 반환합니다.
     - **`String getPattern()`**: 잘못된 패턴을 반환합니다.
     - **`int getIndex()`**: 오류가 발생한 위치의 인덱스를 반환합니다.

### 주요 인터페이스
자바에서는 정규식과 관련된 특정 인터페이스를 제공하지는 않지만, **`CharSequence`** 인터페이스가 정규식 관련 메서드에서 사용됩니다.

- **`CharSequence` 인터페이스**
  - 문자열을 읽기 위한 인터페이스로, `String`, `StringBuilder`, `StringBuffer` 등이 이를 구현합니다.
  - `Pattern`과 `Matcher` 클래스의 입력으로 `CharSequence`를 사용하기 때문에, 문자열과 관련된 다양한 객체들을 정규식에서 활용할 수 있습니다.

### 정규식을 활용한 자바 코드 예제
자바에서 정규식을 사용하기 위해 `Pattern`과 `Matcher` 클래스를 사용하는 일반적인 코드 예시는 다음과 같습니다:

```java
import java.util.regex.Pattern;
import java.util.regex.Matcher;

public class RegexExample {
    public static void main(String[] args) {
        // 정규식 패턴을 컴파일하여 Pattern 객체 생성
        Pattern pattern = Pattern.compile("\\d+");  // 숫자에 매칭되는 정규식
        
        // Matcher 객체 생성
        Matcher matcher = pattern.matcher("There are 123 apples and 456 oranges.");

        // find()를 사용하여 문자열 내에서 숫자를 찾아서 출력
        while (matcher.find()) {
            System.out.println("Found number: " + matcher.group());
            System.out.println("Start index: " + matcher.start());
            System.out.println("End index: " + matcher.end());
        }
    }
}
```

#### 설명
- **`Pattern.compile("\\d+")`**: 숫자에 해당하는 정규식을 컴파일합니다.
- **`Matcher matcher = pattern.matcher("There are 123 apples and 456 oranges.")`**: 입력 문자열을 매칭할 `Matcher` 객체를 생성합니다.
- **`matcher.find()`**: 문자열 내에서 숫자를 찾습니다.
- **`matcher.group()`**: 현재 매칭된 부분 문자열을 반환합니다.
- **`matcher.start()`**, **`matcher.end()`**: 매칭된 부분의 시작 및 끝 위치를 반환합니다.

### `String` 클래스에서 정규식 활용 메서드
`java.lang.String` 클래스에서도 정규식을 활용할 수 있는 여러 메서드를 제공합니다.

1. **`boolean matches(String regex)`**: 문자열이 주어진 정규식과 일치하는지 여부를 반환합니다.
   ```java
   String str = "12345";
   boolean isMatch = str.matches("\\d+");  // true
   ```

2. **`String[] split(String regex)`**: 정규식을 기준으로 문자열을 나누어 배열로 반환합니다.
   ```java
   String str = "apple,banana,orange";
   String[] fruits = str.split(",");  // ["apple", "banana", "orange"]
   ```

3. **`String replaceAll(String regex, String replacement)`**: 정규식과 일치하는 모든 부분 문자열을 주어진 대체 문자열로 변경합니다.
   ```java
   String str = "123abc456";
   String result = str.replaceAll("\\d", "*");  // "******abc******"
   ```

4. **`String replaceFirst(String regex, String replacement)`**: 정규식과 일치하는 첫 번째 부분 문자열을 주어진 대체 문자열로 변경합니다.
   ```java
   String str = "123abc456";
   String result = str.replaceFirst("\\d", "*");  // "*23abc456"
   ```

### 자주 사용하는 정규식 관련 메서드 정리
- **`Pattern.compile(String regex)`**: 정규식 패턴을 컴파일하여 `Pattern` 객체를 생성.
- **`Matcher matcher(String input)`**: 문자열에 대해 패턴 매칭을 수행하기 위해 `Matcher` 객체를 생성.
- **`Matcher.matches()`**: 전체 문자열이 정규식과 일치하는지 여부를 검사.
- **`Matcher.find()`**: 문자열 내에 정규식과 일치하는 부분이 있는지 검사.
- **`Matcher.group()`**: 매칭된 부분 문자열을 반환.
- **`String.matches(String regex)`**: 문자열이 정규식과 일치하는지 확인.
- **`String.split(String regex)`**: 정규식을 기준으로 문자열을 나눠 배열로 반환.
- **`String.replaceAll(String regex, String replacement)`**: 정규식과 일치하는 모든 부분 문자열을 변경.

### 결론
자바에서는 정규식을 다루기 위해 `Pattern`과 `Matcher` 클래스를 주요 도구로 사용합니다. 이 클래스들은 문자열 매칭, 분리, 변환 등을 손쉽게 수행할 수 있게 도와줍니다. `String` 클래스에서도 정규식을 활용하는 다양한 메서드를 제공하여 간편하게 정규 표현식을 사용할 수 있습니다. 이러한 클래스와 메서드를 잘 이해하고 활용하면 복잡한 문자열 처리를 보다 쉽게 구현할 수 있습니다.

---

정규 표현식(Regular Expression, Regex)은 문자열에서 특정 패턴을 찾거나, 매칭하거나, 변환하는 데 사용되는 도구입니다. 자바뿐만 아니라 여러 프로그래밍 언어에서 사용되며, 매우 강력한 문자열 처리 기능을 제공합니다. 정규 표현식에는 다양한 **메타문자**와 **기호**가 있으며, 이를 통해 문자열의 특정 패턴을 정의할 수 있습니다. 자주 사용하는 주요 표현들을 정리해 보겠습니다.

### 1. 기본 메타문자
정규식에서 **메타문자**는 특별한 의미를 가진 문자들입니다. 문자열 매칭을 정의하는 데 사용됩니다.

- **`.` (마침표)**: 임의의 문자 한 개를 의미합니다.
  - 예: `"a.c"`는 `"abc"`, `"a1c"`와 매칭됩니다. (단, 개행 문자는 제외)
  
- **`*` (별표)**: 바로 앞의 문자가 0번 이상 반복됨을 의미합니다.
  - 예: `"a*"`는 `""`, `"a"`, `"aa"`, `"aaa"`와 매칭됩니다.

- **`+` (더하기)**: 바로 앞의 문자가 1번 이상 반복됨을 의미합니다.
  - 예: `"a+"`는 `"a"`, `"aa"`, `"aaa"`와 매칭됩니다.

- **`?` (물음표)**: 바로 앞의 문자가 0번 또는 1번 등장함을 의미합니다.
  - 예: `"a?"`는 `""`, `"a"`와 매칭됩니다.

- **`|` (파이프)**: OR 연산을 의미합니다.
  - 예: `"a|b"`는 `"a"` 또는 `"b"`와 매칭됩니다.

- **`^` (캐럿)**: 문자열의 시작을 의미합니다.
  - 예: `"^a"`는 `"a"`로 시작하는 문자열과 매칭됩니다.

- **`$` (달러)**: 문자열의 끝을 의미합니다.
  - 예: `"a$"`는 `"a"`로 끝나는 문자열과 매칭됩니다.

### 2. 문자 클래스
문자 클래스는 대괄호 `[]` 안에 매칭하고자 하는 문자를 나열하여 사용합니다.

- **`[abc]`**: `a`, `b`, `c` 중 하나와 매칭됩니다.
  - 예: `"b"`는 `[abc]`와 매칭됩니다.

- **`[a-z]`**: 소문자 `a`부터 `z`까지 중 하나와 매칭됩니다.
  - 예: `"g"`는 `[a-z]`와 매칭됩니다.

- **`[A-Z]`**: 대문자 `A`부터 `Z`까지 중 하나와 매칭됩니다.

- **`[0-9]`**: 숫자 `0`부터 `9`까지 중 하나와 매칭됩니다.

- **`[^abc]`**: `a`, `b`, `c`를 제외한 나머지 문자와 매칭됩니다. `^`는 부정을 의미합니다.

### 3. 자주 사용하는 기호
- **`\d`**: 숫자(`0-9`)와 매칭됩니다.
  - 예: `"123"`은 `\d+`와 매칭됩니다.

- **`\D`**: 숫자가 아닌 문자와 매칭됩니다.
  - 예: `"abc"`는 `\D+`와 매칭됩니다.

- **`\w`**: 단어 문자(알파벳 대소문자, 숫자, 언더스코어 `_`)와 매칭됩니다.
  - 예: `"a1_"`은 `\w+`와 매칭됩니다.

- **`\W`**: 단어 문자가 아닌 문자와 매칭됩니다.
  - 예: `"!"`는 `\W`와 매칭됩니다.

- **`\s`**: 공백 문자(스페이스, 탭, 개행 등)와 매칭됩니다.
  - 예: `" "`는 `\s`와 매칭됩니다.

- **`\S`**: 공백 문자가 아닌 문자와 매칭됩니다.
  - 예: `"a"`는 `\S`와 매칭됩니다.

### 4. 수량자
수량자는 패턴의 반복 횟수를 정의합니다.

- **`{n}`**: 정확히 `n`번 반복됨을 의미합니다.
  - 예: `"a{3}"`는 `"aaa"`와 매칭됩니다.

- **`{n,}`**: 최소 `n`번 이상 반복됨을 의미합니다.
  - 예: `"a{2,}"`는 `"aa"`, `"aaa"`, `"aaaa"` 등과 매칭됩니다.

- **`{n,m}`**: 최소 `n`번, 최대 `m`번 반복됨을 의미합니다.
  - 예: `"a{2,4}"`는 `"aa"`, `"aaa"`, `"aaaa"`와 매칭됩니다.

### 5. 그룹화와 역참조
- **`()` (소괄호)**: 그룹을 지정하며, 매칭된 그룹을 캡처할 수 있습니다.
  - 예: `"(abc)"`는 `"abc"`와 매칭되며, 그룹으로 캡처됩니다.

- **`\1`, `\2` 등**: 캡처된 그룹을 참조할 때 사용됩니다.
  - 예: `"(\\d)\\1"`는 동일한 숫자가 연속해서 두 번 나오는 경우와 매칭됩니다. 예를 들어, `"11"`은 매칭됩니다.

### 6. 앵커 (Anchors)
앵커는 특정 위치를 지정합니다.

- **`^`**: 문자열의 시작 부분을 나타냅니다.
  - 예: `"^Hello"`는 문자열이 `"Hello"`로 시작해야 매칭됩니다.

- **`$`**: 문자열의 끝 부분을 나타냅니다.
  - 예: `"world$"`는 문자열이 `"world"`로 끝나야 매칭됩니다.

- **`\b`**: 단어 경계를 나타냅니다.
  - 예: `"\bword\b"`는 `"word"`라는 단어가 단독으로 존재할 때 매칭됩니다.

- **`\B`**: 단어 경계가 아닌 위치를 나타냅니다.
  - 예: `"\Bword"`는 `"sword"`와 같이 `"word"` 앞에 다른 문자가 있을 때 매칭됩니다.

### 7. 예시를 통한 이해
아래는 위의 주요 표현들을 사용한 간단한 예시입니다.

- **`\\d{3}-\\d{2}-\\d{4}`**: 세 자리 숫자, 두 자리 숫자, 네 자리 숫자로 구성된 패턴으로, 주로 미국의 사회 보장 번호 형식과 매칭됩니다. 예: `"123-45-6789"`.

- **`[A-Z][a-z]*`**: 대문자로 시작하고 그 뒤에 소문자가 0번 이상 반복되는 단어를 매칭합니다. 예: `"Hello"`, `"Apple"`.

- **`\\w+@\\w+\\.\\w+`**: 이메일 주소 형식과 매칭합니다. 예: `"test@example.com"`.

### 결론
정규 표현식은 다양한 패턴을 사용하여 문자열의 복잡한 조건을 매칭하고 검증하는 데 매우 유용합니다. 위에서 다룬 주요 메타문자와 기호들은 자주 사용되는 기본적인 것들이며, 실제로는 이를 조합하여 더 복잡한 패턴을 만들 수 있습니다. 정규식을 잘 이해하고 사용하는 것은 문자열 처리의 강력한 무기가 될 수 있습니다.
